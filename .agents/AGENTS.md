# 📈 주식 프로젝트 커스텀 규칙 (AI 행동 강령)

이 문서는 저(AI)가 이 프로젝트에서 작업할 때 0순위로 지켜야 하는 **절대 원칙**을 모아둔 곳입니다. 고객님의 지시에 따라 규칙은 계속 추가 및 정리됩니다.

---

## 📌 확정된 5대 핵심 개발 원칙

### 1. 철저한 검증 (눈으로 확인하기 전엔 넘겨짚지 않기)
* 절대 '감'이나 추측으로 코드를 수정하지 않습니다. 
* 에러가 발생하거나 로직을 짤 때는, 반드시 실제 데이터나 백그라운드 로그를 직접 스캔하여 정확한 원인을 파악한 뒤에만 코드를 작성합니다.

### 2. 1-Click 실행 원칙 (고객님께 귀찮은 작업 요구 금지)
* 복잡한 명령어, 터미널 세팅, 패키지 설치 등은 모두 AI가 백그라운드에서 알아서 처리합니다.
* 고객님은 오직 바탕화면의 `실행하기.bat` 더블클릭 한 번만 하시면 모든 것이 돌아가도록 환경을 완벽하게 자동화합니다.

### 3. 클린 UI 유지 (시각적 쓰레기 노출 금지)
* 웹페이지와 터미널 창에는 오직 '꼭 필요한 정보'만 깔끔하게 띄웁니다.
* 에러 메시지나 쓸데없는 디버깅용 텍스트(`[0101]`, `잔고확인` 등)가 사용자 화면에 노출되지 않도록 강력하게 필터링합니다. 디버깅 로그는 숨겨진 `debug` 폴더에만 조용히 남깁니다.

### 4. 스마트한 버전 관리 (Git 자동화)
* 무식하게 폴더를 복사해서 백업하지 않습니다.
* 기능 개발이나 버그 수정 등 의미 있는 단위의 작업이 끝날 때마다, AI가 알아서 `git commit`과 `git push`를 실행하여 깃허브(GitHub)에 코드를 깔끔하게 영구 보존합니다.

### 5. 완벽한 보안 유지 (API 키 관리)
* 모든 API 키(DART, 키움 등)와 비밀번호는 오직 `.env` 파일 안에서만 관리합니다.
* 어떤 경우에도 소스 코드(.py, .html 등)에 키 값을 직접 하드코딩하지 않으며, 깃허브에 `.env` 파일이 실수로 올라가지 않도록 철저히 차단합니다.
