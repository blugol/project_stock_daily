# HTS 연동 트러블슈팅 및 문제 해결 기록

## 🚨 발생했던 문제 현상
1. **고정된 종목(0101)만 뜨는 현상**: HTS에서 종목을 변경해도 웹페이지에서 계속 `[0101]` 또는 엉뚱한 화면 이름만 고정되어 나타남.
2. **"잔고확인" 등 시스템 텍스트가 뜨는 현상**: 종목명 대신 `잔고확인(실시간)` 등 원치 않는 시스템 창의 텍스트가 종목명으로 오인되어 출력됨.
3. **아무것도 연동되지 않는 현상**: 종목을 아무리 바꿔도 종목명이 아예 웹페이지로 전송되지 않음.

## 🔍 원인 분석
### 1. HTS 화면 구조에 대한 오해
초기 개발 단계에서는 창 제목이 `[0101] 키움현재가 - 삼성전자`처럼 **항상 하이픈(-)과 종목명이 같이 붙어있는 하나의 윈도우**라고 가정했습니다.
그러나 실제 고객님의 HTS 환경을 분석한 결과 다음과 같은 사실을 발견했습니다:
* 화면 구성에 따라 부모 창(예: `[0600] 종합차트` 등)과 종목을 표시하는 **자식 창(예: `코스모화학`)이 아예 분리**되어 있었습니다.
* 즉, 종목명 텍스트 자체(`솔루엠`, `코스모화학` 등)가 통째로 하나의 독립적인 창 텍스트를 이루고 있었으며, **하이픈(-)이 아예 존재하지 않는 구조**였습니다.

### 2. 포커스(Focus) 방식의 한계
클릭한 창의 포커스(Focus)를 쫓아가서 텍스트를 읽어오도록 코드를 짰으나, 사용자가 `관심종목` 그리드나 `잔고확인` 창을 클릭할 경우 그 창이 포커스를 가져가게 되므로 `잔고확인(실시간)` 같은 텍스트가 잡히는 부작용이 발생했습니다.
실제 종목 연동은 '관심종목'을 클릭하면 '차트창'이 뒤에서 알아서 바뀌는 구조(백그라운드 연동)인데, 이를 포커스 방식으로는 캐치할 수 없었습니다.

## 💡 해결 방법 (최종 적용된 로직)
1. **텍스트 100% 감시 및 변화(Diff) 추적 방식으로 변경**
   * 영웅문 안에 떠 있는 **모든 화면의 텍스트를 통째로 감시**합니다.
   * 사용자가 종목을 클릭해서 다른 창(차트나 호가창 등)의 텍스트가 기존에서 새로운 텍스트로 **"변경(Diff)"**되는 순간을 캐치합니다.
2. **하이픈(-) 의존성 완전 제거**
   * 하이픈이 있는 경우(`- 삼성전자`)와 하이픈이 아예 없는 쌩텍스트(`삼성전자`) 모두를 포용할 수 있도록 로직을 분리했습니다.
3. **초강력 필터링 도입 (화이트리스트/블랙리스트 방식)**
   * `IGNORE_EXACT`: "매수", "매도", "수량", "호가", "메뉴툴바" 등 종목명이 될 수 없는 HTS 시스템의 모든 단어를 모아 원천 차단했습니다.
   * `IGNORE_KEYWORDS`: "잔고", "실시간", "계좌" 등 특정 단어가 포함되면 아예 무시하도록 필터를 강화했습니다.
   * 시간표시, 100단위 숫자, 단축키 표시 등을 정규식으로 걸러내 오직 **유효한 종목명**만 통과되도록 방어 코드를 작성했습니다.

## ✅ 결론
결과적으로 HTS 내부 텍스트 변화를 실시간으로 탐지하면서도, 가짜 종목명을 완벽히 걸러내는 알고리즘이 적용되어 **고객님이 어떤 화면 번호나 세팅을 쓰더라도 100% 연동**되는 안정적인 코드가 완성되었습니다.
