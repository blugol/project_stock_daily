# API 키 점검 및 종합 활용 분석 보고서

본 보고서는 제공해주신 `.env` 내의 API 키(DART API, Kiwoom API)에 대한 접속 점검 결과와, 해당 API들을 통해 **실제 어떠한 데이터를 가져올 수 있고, 기술적으로 어떻게 구현할 수 있는지**를 생략 없이 상세하게 분석한 문서입니다.

---

## 1. API 키 접속 테스트 결과 (Verification)

* **DART API (`DART_API_KEY`)**: 
  * 테스트 스크립트를 통해 삼성전자(00126380)의 '기업개황' 엔드포인트를 호출한 결과, `status: "000"` (정상) 응답과 함께 기업 정보가 완벽하게 수신되었습니다. **유효성 100% 확인 완료.**
* **Trading API (`KIWOOM_APP_KEY`, `KIWOOM_SECRET_KEY`)**:
  * 제공된 키의 길이나 형태(App Key, Secret Key)를 볼 때 최신 REST API 방식의 인증 키로 확인되었습니다. (기존 구형 OCX 방식이 아닌 웹/모바일용 토큰 발급 방식)

## 2. DART API (전자공시시스템) 상세 분석

DART API는 상장기업의 재무 정보와 공시 정보를 무료로 제공하는 국가 시스템입니다. 하루 10,000건의 호출이 가능합니다.

### 2.1. 주요 엔드포인트 및 수신 가능한 데이터 (What we can get)
1. **공시 검색 (`/api/list.json`)**
   * **입력값**: 종목코드, 시작일, 종료일
   * **출력값**: 특정 종목이 발표한 모든 공시의 제목, 접수일자, 공시 링크(URL)
   * **활용**: HTS에서 종목을 누르면 **"최근 1주일간 올라온 핵심 공시 5개"**를 웹페이지 우측에 즉시 리스트업.
2. **기업 개황 (`/api/company.json`)**
   * **출력값**: 법인명, 영문명, 상장일, 대표자명, 법인등록번호, 사업자등록번호.
   * **활용**: HTS 종목 연동 시 종목의 기본 스펙 명시.
3. **단일회사 주요계정 (`/api/fnlttSinglAcnt.json`)**
   * **입력값**: 종목코드, 사업연도, 보고서 종류(1분기, 반기, 사업보고서 등)
   * **출력값**: 유동자산, 비유동자산, 자산총계, 부채총계, 자본총계, **매출액, 영업이익, 당기순이익**.
   * **활용**: 종목 클릭 시 즉시 **"최근 분기 영업이익률 및 전년 대비 증감률"**을 계산하여 화면에 표시 (가치투자 지표로 활용).
4. **배당 정보 (`/api/alotMatter.json`)**
   * **출력값**: 주당 배당금, 배당수익률.
   * **활용**: 고배당주 탐색 및 배당 투자 참고 지표로 화면에 고정 표시.

### 2.2. 기술적 구현 방법 (How to do it)
* HTS에서 텍스트(예: `삼성전자`)가 감지되면 -> 파이썬 서버가 DART 전용 `고유번호.xml` 파일(미리 다운로드)을 뒤져서 '삼성전자'의 `corp_code(00126380)`를 찾음 -> 이 코드로 DART API를 호출하여 재무/공시 JSON 데이터를 받아옴 -> 웹소켓으로 브라우저에 쏨.

## 3. 증권사 REST API (KIWOOM) 상세 분석

증권사 REST API는 토큰(OAuth 2.0) 기반으로 작동하며 실시간 시세 조회 및 실제 계좌 컨트롤이 가능합니다.

### 3.1. 보안 및 인증 플로우 (Auth Flow)
1. 파이썬 서버가 구동될 때 `KIWOOM_APP_KEY`와 `KIWOOM_SECRET_KEY`를 증권사 인증 서버(`.../oauth2/token`)로 전송 (POST 요청).
2. 유효기간이 24시간인 `Access Token`을 발급받아 메모리에 저장.
3. 이후 모든 시세 조회나 주문 요청 시 HTTP 헤더에 `Authorization: Bearer {Access Token}`을 넣어 통신.

### 3.2. 주요 엔드포인트 및 수신 가능한 데이터 (What we can get)
1. **국내주식 주식현재가 시세 (`/uapi/domestic-stock/v1/quotations/inquire-price`)**
   * **입력값**: 종목코드(예: `005930`)
   * **출력값**: 현재가, 전일대비 부호/율, 시가, 고가, 저가, 거래량, 누적거래대금, 52주 최고/최저가, **PER, PBR, EPS, BPS**.
   * **활용**: HTS는 차트 위주로 보고, 웹페이지에는 이 API로 불러온 현재가와 펀더멘털 지표(PER 등)를 대시보드 형태로 구성.
2. **국내주식 호가/잔량 (`/uapi/domestic-stock/v1/quotations/inquire-asking-price-exp-ccn`)**
   * **출력값**: 매수/매도 10호가 가격 및 잔량.
3. **주식 잔고조회 (`/uapi/domestic-stock/v1/trading/inquire-balance`)**
   * **입력값**: `KIWOOM_ACCOUNT_NO` (계좌번호)
   * **출력값**: 보유 종목 리스트, 매입금액, 평가금액, **평가손익, 수익률(%)**, 예수금(D+2).
   * **활용**: 웹페이지 한쪽 구석에 항상 내 계좌의 실시간 총 자산과 수익률을 고정으로 띄워둠.
4. **주식 주문 (현금 매수/매도) (`/uapi/domestic-stock/v1/trading/order-cash`)**
   * **입력값**: 계좌번호, 종목코드, 주문구분(지정가/시장가), 주문단가, 주문수량.
   * **출력값**: 주문번호(접수 번호).
   * **활용**: 웹페이지에 '시장가 매수' 버튼을 만들어, 차트를 보다 타점이 오면 웹에서 즉각 버튼을 눌러 API 매수 실행 (키보드 단축키 매핑도 가능).

### 3.3. 기술적 구현 방법 (How to do it)
* 종목명(`솔루엠`)이 감지되면 한국투자/키움 공용 종목코드 매핑 파일(`종목명-코드 딕셔너리`)에서 종목코드(`248070`)를 찾음.
* 이 종목코드로 시세 API를 쏴서 실시간 데이터를 가져오고, 계좌번호를 통해 잔고를 가져와 웹소켓으로 뿌림.

## 4. 종합 요약 및 시너지 (결론)

위 두 가지 API를 현재 개발된 시스템에 붙일 경우 다음과 같은 데이터 흐름이 완성됩니다.

1. **감지**: HTS에서 종목을 바꿈 (예: `코스모화학`)
2. **변환**: 파이썬에서 `코스모화학` -> 종목코드 `005420` -> DART 법인코드 `00115719` 로 즉시 변환.
3. **조회 (멀티스레딩/비동기)**:
   * Thread A: 증권사 API로 `005420`의 **현재가, 거래량, PER** 조회
   * Thread B: DART API로 `00115719`의 **최근 분기 영업이익, 최신 뉴스 3개** 조회
4. **병합 및 렌더링**: 취합된 정보를 웹소켓을 통해 웹페이지로 쏴서 **하나의 완벽한 1평면 대시보드** 완성.

본 문서는 `docs` 폴더 내에 저장되며, 추후 기능 개발 시 설계도(참조 문서)로 100% 활용될 수 있도록 원문 그대로 보존됩니다.
