# HTS 실시간 종목 연동 시스템 (HTS Real-time Stock Tracker)

## 📌 프로젝트 개요
키움증권 영웅문4 HTS에서 사용자가 보고 있는 종목을 실시간으로 추적하여 웹페이지에 연동하는 시스템입니다. Open API의 직접적인 화면 연동 기능 부재를 극복하기 위해 윈도우 UI 텍스트 변화를 감지하는 우회 방식을 사용했습니다.

---

## 🛠 기술 스택 (Tech Stack)
* **Backend**: Python 3
  * `win32gui`, `win32process`: Windows API를 이용한 HTS 화면 텍스트 실시간 스크래핑
  * `websockets`: 웹페이지와의 실시간 양방향 데이터 통신
  * `asyncio`: 비동기 웹소켓 서버 및 스케줄링 처리
  * `re`: 정규 표현식을 이용한 텍스트 파싱
* **Frontend**: HTML5, Vanilla JavaScript
  * `WebSocket API`: 파이썬 서버로부터 실시간 종목명 수신 및 DOM 업데이트

---

## ⚙️ 워크플로우 (Workflow)
1. **서버 실행**: `start.bat`을 통해 파이썬 웹소켓 서버(`server.py`)와 프론트엔드(`index.html`)가 동시에 실행됩니다.
2. **윈도우 탐색 (0.3초 주기)**: 서버가 백그라운드에서 '영웅문' 텍스트를 포함하는 최상위 창과 그 내부의 모든 자식 창(MDI Child)을 스캔합니다.
3. **텍스트 Diff 감지**: 각 자식 창의 텍스트가 이전 0.3초 전의 상태와 달라졌는지 비교합니다.
4. **유효성 검증**: 변경된 텍스트가 순수 종목명인지 판별합니다.
   * `IGNORE_EXACT` 해시셋을 통해 "매수", "매도", "수량" 등의 시스템 단어 필터링
   * 정규식을 통해 숫자, 시간, 좌표 등 의미 없는 데이터 차단
5. **데이터 전송**: 검증을 통과한 새로운 종목명을 웹소켓 클라이언트(브라우저)로 Push 합니다.
6. **UI 업데이트**: 브라우저는 수신된 종목명을 화면의 `<h1>` 태그에 즉시 렌더링합니다.

## 📝 오늘 개발 내용 및 주요 이슈 해결 (Troubleshooting)

### 1. 개발 내용
* 로컬 환경에서 실행 가능한 웹소켓 기반의 실시간 종목명 렌더러 구축 완료
* 원클릭 실행을 위한 `start.bat` 및 파일 정리(`app` 폴더화) 완료

### 2. 주요 이슈 및 해결 과정
* **[이슈 1] 백그라운드 창 텍스트 고정 문제 (0101 고정 현상)**
  * **원인**: 처음 발견한 특정 화면(예: `[0101]`)만 고정적으로 읽어오는 로직 결함.
  * **해결**: 포커스 기반 추적 대신 **"모든 창의 텍스트 변경(Diff)을 실시간으로 감지"**하는 방식으로 전면 개편.

* **[이슈 2] 종목명 추출 실패 (하이픈 부재 문제)**
  * **원인**: HTS 화면 제목이 항상 `[화면번호] 화면명 - 종목명` 형태일 것이라 가정했으나, 실제 고객의 UI 세팅에서는 종목명이 하이픈 없이 단독 텍스트(예: `솔루엠`)로 존재하는 창들이 발견됨.
  * **해결**: 하이픈 의존성을 제거하고 변경된 텍스트 자체가 유효한 종목명인지 검사하는 화이트리스트/블랙리스트 기반 필터 알고리즘으로 교체.

* **[이슈 3] "잔고확인", "메뉴툴바" 등 시스템 텍스트 혼입**
  * **원인**: 포커스를 쫓아가거나 텍스트 변화를 감지할 때, 종목과 상관없는 시스템 UI 요소의 텍스트가 잡힘.
  * **해결**: `IGNORE_EXACT` 리스트 및 정규식을 강력하게 적용하여 종목이 될 수 없는 키워드를 철저히 차단.

* **[이슈 4] 배치 파일 인코딩 오류**
  * **원인**: `start.bat` 파일 내에 작성된 한글 주석이 윈도우 `cmd.exe`의 ANSI(CP949) 해석 과정에서 깨지며 오류 발생.
  * **해결**: 배치 파일 내의 모든 한글 주석 및 텍스트를 영문으로 대체하여 인코딩 의존성 완벽 제거 및 터미널 1개로 단일화.
